Welcome to my personal space

This self-hosted webpage is brought to you by the power of light ☀️!

Run on the Wemos D1 Mini v2.2.0 (utilizing the ESP8266 chip), powered by a 10W solar panel, with 1900mAh battery for storing the gathered energy, this is truly a self-sufficient solution intended to minimize the carbon footprint of the modern technology.

Due to the unpredictability of the weather, I kindly ask for your understanding regarding the availability of this website. It's not always sunny here in Poland! If you ever encounter any problems regarding access, please try again later.

Programming experience

I won't lie that I'm a full-stack developer. I'm certainly not, despite my desire to be that good.

I'm mainly a Python backend developer enthusiast with some knowledge regarding devops and frontend development (check my CV for more details).

I try to maintain high code health via usage of external tools for type checking (MyPy), code style (Flake8) and full test coverage.

I prefer to use object-oriented programming approach where possible as in my opinion it's easier to read and understand.

I have strong drive to write generic code that can be easily extended with new features.

I hate copy-paste code and refactor it whenever possible.

I like to use long descriptive names for the variables and functions and avoid any abbreviations.

I try to refrain from writing code comments apart from docstrings, as I believe they get outdated too fast.

Work culture

When it comes to work I believe that a good communication is a key to success. I think that most of the problems regarding software not meeting the needs of a client are caused by the lack of proper discussion and not enough willingness to understand each other. It's much easier for me to work when as much information regarding the requirements is available prior to starting development. I try to write down all of those in form of "acceptance criteria" and ensure that they are thoroughly tested before marking my job as done.

Personally I'm a huge fan of the SCRUM methodology - it naturally fits my way of thinking about software development. I like taking parts in discussions that try to refine the process of managing the project. I really like dividing huge tasks into smaller, more manageable pieces. I prefer daily commits to huge features that take weeks to finish. I despise checking huge PRs with multiple changes unrelated to each other. I'm always eager for some pair programming - in my experience this helps tremendously with unloading the bus factor, and usually results in a better code written.

I believe that in order to get the best team performance people need to get to know each other. Integration trips, or any other activities encouraging exchange of views (not necessarily related to work) are a great way of maintaining healthy relations between team members.
